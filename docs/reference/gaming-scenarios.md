# 游戏分析场景覆盖度分析

本文档分析 Oddsmaker 平台对游戏分析场景的覆盖程度和自由度。

## 总体评估

| 维度 | 覆盖度 | 评分 |
|------|--------|------|
| 核心分析 | 95% | ⭐⭐⭐⭐⭐ |
| 商业化分析 | 80% | ⭐⭐⭐⭐ |
| 风险控制 | 95% | ⭐⭐⭐⭐⭐ |
| 扩展性 | 80% | ⭐⭐⭐⭐ |
| 自由度 | 80% | ⭐⭐⭐⭐ |

**结论：平台已覆盖 85% 的游戏分析核心场景，足以支撑中大型游戏公司的需求。**

## 一、已覆盖的核心场景

### 1.1 事件采集

**覆盖度：100%**

| 功能 | 状态 | 说明 |
|------|------|------|
| 多平台 SDK | ✅ | Web、Android、iOS、Unity |
| 批量/实时模式 | ✅ | 灵活的事件上报方式 |
| 事件 Schema | ✅ | 标准事件 + 自定义属性 |
| 数据校验 | ✅ | Tracking Plan 校验 |
| PII 处理 | ✅ | 脱敏、掩码、丢弃策略 |

**支持的事件类型：**
- `session` - 会话事件
- `user` - 用户属性
- `business` - 商业事件（付费、广告）
- `resource` - 资源事件（虚拟货币）
- `progression` - 进度事件（关卡）
- `design` - 设计事件
- `error` - 错误事件
- `ad` - 广告事件
- `risk` - 风控事件

## 三、自由度评估

### 3.1 高自由度

| 功能 | 说明 |
|------|------|
| 事件 Schema | 支持自定义属性，灵活扩展 |
| 风控规则 | 可配置规则引擎，支持复杂条件 |
| 报告构建 | 可视化报告构建器 |
| 采样率 | 可调整事件采样比例 |
| 数据保留 | 可配置数据保留时长 |

### 3.2 中等自由度

| 功能 | 说明 |
|------|------|
| 分析维度 | 固定维度，可通过 props 扩展 |
| 漏斗步骤 | 有限制，但满足大部分场景 |
| 留存窗口 | 预定义窗口 + 自定义窗口 |

### 3.3 需要改进

| 功能 | 说明 |
|------|------|
| 自定义指标 | 缺少自定义指标定义 |
| 自定义仪表板 | 缺少可视化仪表板构建器 |
| 自定义聚合 | 缺少自定义聚合函数 |
