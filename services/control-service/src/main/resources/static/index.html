控制台占位页

当前控制面的真实模型已经固定为 单公司部署、 多游戏、 多环境。 这个静态页只保留最小说明，等待正式控制台替换。

Oddsmaker 的目标边界是 单公司部署、 多游戏、 多环境，公开管理接口应围绕 Game -> Environment -> Storage Profile 展开。

当前约束

  • 控制面资源围绕 Game / Environment / API Key / Risk Rule 组织。
  • Environment 表达逻辑发布阶段，Storage Profile 表达物理数据路由。
  • API Key 绑定到游戏和环境，而不是公司租户。
  • 风控策略、PII 策略、Tracking Plan 都应按游戏环境下发。

推荐接口

  • POST /api/games 创建游戏
  • GET /api/games/{gameId}/environments 查询环境
  • GET /api/storage-profiles 查询存储路由配置
  • POST /api/api-keys 创建 API Key
  • GET /api/api-keys?gameId=&environmentId= 查询 API Key

请求示例

{
  "name": "Poker Mobile",
  "displayName": "Poker Mobile",
  "genre": "RPG",
  "platforms": ["MOBILE"]
}

  • 实验与配置接口统一使用 gameId + environmentId 作为逻辑边界。
