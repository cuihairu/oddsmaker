控制台占位页

当前控制面的真实模型已经固定为 单公司部署、 多游戏、 多环境。 这个静态页只保留最小说明，等待正式控制台替换。

Oddsmaker 的目标边界是 单公司部署、 多游戏、 多环境，公开管理接口应围绕 gameId + environmentId 展开。

当前约束

  • 控制面资源围绕 Game / Environment / API Key / Risk Rule 组织。
  • API Key 绑定到游戏和环境，而不是公司租户。
  • 风控策略、PII 策略、Tracking Plan 都应按游戏环境下发。

推荐接口

  • POST /api/keys 创建 API Key
  • GET /api/keys?gameId=&environmentId= 查询 API Key
  • GET /api/keys/{apiKey} 获取策略明细
  • PUT /api/keys/{apiKey}/policy 更新策略

请求示例

{
  "gameId": "poker-mobile",
  "environmentId": "prod",
  "name": "client-prod-key"
}

后续工作

  • 实验与配置接口统一使用 gameId + environmentId。
  • 控制面权限模型改成公司内按游戏和环境授权。
